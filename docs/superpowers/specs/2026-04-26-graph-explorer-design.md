# Graph Explorer — Design

**Date:** 2026-04-26
**Status:** Draft, awaiting review

## Goal

Build a comprehensive interactive 3D explorer over the Kuzu knowledge graph that lets a human (a) eyeball every Product, ESHACode, ProductCategory, and ESHACategory together in one shared embedding space, and (b) emits a ranked CSV of suspect Product → ESHACode mappings into the existing quarantine pipeline.

The explorer makes "popcorn floating in the milk cluster" visually obvious, and the suspect CSV makes it actionable through the heal/quarantine machinery already in place.

## Non-goals

- Replacing the existing `taxonomy_conflict_map.html` or `self_heal_conflict_map.html`. They stay; they're scoped to specific diagnostic slices. This is the whole-corpus explorer.
- Re-architecting the graph schema. We read from Kuzu and write outputs alongside the existing pass artifacts. No schema changes.
- Auto-applying any remappings. The viewer surfaces; the existing pass machinery decides and applies.
- Building a server. Everything is static HTML + bundled JSON, regenerated by a Python script.

## Assumptions (please confirm or override)

1. **Embedding model:** local `sentence-transformers/all-MiniLM-L6-v2` (free, ~5 min for full corpus, no API key, ships in repo).
2. **Scope of v1:** full corpus from the start, not a single-family pilot.
3. **Ingredients:** v1 uses the existing `Token` node table as the lexical signal. Adding a true `Ingredient` node table is deferred to a follow-up (the existing `graph/ingest/build_ingredients.py` may already do this — to be confirmed when the implementing plan inspects it).

## Architecture

Three new files, one regeneration command:

```
graph/queries/build_graph_explorer.py    # orchestrator, emits everything below
graph/queries/embed_nodes.py             # node → vector module (importable)
graph/queries/score_suspicion.py         # cross-view scoring module (importable)
implementation/output/graph_explorer/
    index.html                           # the viewer
    bundle.js                            # Three.js + deck.gl + app code
    data.json                            # nodes, edges, embeddings, scores
graph/quarantine/embedding_suspects.csv  # suspect Product→ESHA mappings
```

Run with: `python3 graph/queries/build_graph_explorer.py`

Open with: `open implementation/output/graph_explorer/index.html`

## Components

### 1. `embed_nodes.py`

**Purpose:** Turn each node in the graph into a fixed-length vector.

**Interface:**
```python
def embed_all(conn: kuzu.Connection, model_name: str = "all-MiniLM-L6-v2") -> dict[str, np.ndarray]
# returns {"Product:<gtin_upc>": vec, "ESHACode:<code>": vec, ...}
```

**What it embeds:**
- `Product`: `description` + space-joined `Token` values from `HAS_TOKEN` edges
- `ESHACode`: `code + " " + description`
- `ProductCategory`: `name`
- `ESHACategory`: `name`

**Dependencies:** `kuzu`, `sentence-transformers`, `numpy`. Cached to `graph/cache/node_embeddings.npz` keyed by model name + a content hash, so re-running is cheap.

### 2. `score_suspicion.py`

**Purpose:** Compute, per Product, a suspicion score for its assigned ESHACode and identify the kind of disagreement.

**Interface:**
```python
def score_all(conn, embeddings) -> pd.DataFrame
# columns: gtin_upc, assigned_code, assigned_rank, top1_code, top1_score,
#          text_view_code, graph_view_code, label_view_code, disagreement_kind, suspicion
```

**The three views (per Product P, currently mapped to ESHACode A):**
- **Text view** — cosine of `embed(P.description + tokens)` against every ESHACode embedding. Top-1 = text's vote.
- **Graph view** — average of embeddings of P's neighbors that already have ESHACode mappings, restricted to its `ProductCategory` and shared `Brand`. Cosine that against every ESHACode. Top-1 = graph's vote.
- **Label view** — cosine of `embed(A.code + A.description)` against `embed(P.description + tokens)`. Just measures: does the assigned code's text actually describe the product?

**Suspicion score:** weighted blend, but driven primarily by `assigned_rank` of the assigned code in the text view's ranking. Anything outside top-5 is highly suspect. We also boost suspicion when text and graph views agree with each other but disagree with the label view (the strongest "the assignment is wrong" signal).

**Disagreement kind:** one of `agree | wrong_label | bad_upstream | garbage`, derived from which views agree:
- `agree` → all three views point at A → not suspect
- `wrong_label` → text + graph agree on B, label says A → most likely a wrong ESHA assignment
- `bad_upstream` → text and graph disagree → ingredient parse or category problem
- `garbage` → all three disagree → quarantine candidate

### 3. `build_graph_explorer.py`

**Purpose:** Glue. Connects to Kuzu, calls `embed_nodes` and `score_suspicion`, projects embeddings to 3D (PCA, UMAP), packages everything into `data.json`, copies the static `index.html` and `bundle.js` next to it, writes `embedding_suspects.csv` to `graph/quarantine/`.

**Key data shapes in `data.json`:**
```json
{
  "nodes": [
    {"id": "Product:0123456789012", "kind": "Product", "label": "...",
     "pca": [x,y,z], "umap": [x,y,z],
     "category": "...", "esha_family": "...", "assigned_esha": "...",
     "suspicion": 0.87, "disagreement": "wrong_label", ...},
    {"id": "ESHACode:1004", "kind": "ESHACode", ...},
    ...
  ],
  "edges": [{"from": "...", "to": "...", "kind": "MAPS_TO"}, ...]
}
```

UMAP and PCA are precomputed server-side. Force-directed graph layout is computed client-side at view time using deck.gl over the `edges` array (so it can recompute when filtered).

### 4. The viewer (`index.html` + `bundle.js`)

**Tech:** deck.gl `ScatterplotLayer` (handles 100k+ points smoothly on WebGL2) as the renderer. We do *not* mix in Three.js — your existing viewers use Three.js, but deck.gl is purpose-built for large scatter clouds and has its own scene/camera. Vendored as a single `bundle.js` so the HTML works offline.

**UI layout:**
- Center: 3D scene, drag-to-rotate, scroll-zoom
- Top toolbar: projection toggle (PCA / UMAP / graph-force), color-by dropdown (`esha_family`, `category`, `brand`, `suspicion`, `disagreement_kind`, `assignment_status`), search box (matches against label/id), node-kind toggles (show/hide Products, ESHACodes, etc.)
- Right side panel (when a point is clicked): node label, kind, all metadata, the three view votes side-by-side, top-5 nearest ESHACodes by cosine, list of graph neighbors (clickable to jump)
- Bottom: small histogram of suspicion scores; brushing it filters the cloud

**Key interactions for the audit use case:**
- Set color = `disagreement_kind` → wrong_label points light up red. Brush the suspicion histogram down to high-suspicion only → cloud collapses to just the bad ones. Click any → see why it's flagged.
- Search "popcorn" → all popcorn-related nodes pulse. If any sit visually inside the milk cluster, that's the smoking gun.

## Data flow

```
Kuzu DB ──▶ embed_nodes ──▶ {node_id: vector}
                │
                ├──▶ PCA + UMAP projection ──┐
                │                            ▼
                └──▶ score_suspicion ──▶ data.json ──▶ index.html (viewer)
                              │
                              └──▶ embedding_suspects.csv ──▶ existing pass machinery
```

## Error handling

- **Empty Kuzu DB:** `build_graph_explorer.py` exits with a clear message pointing at `build_kuzu_graph.py`.
- **Missing model:** `embed_nodes.py` downloads `all-MiniLM-L6-v2` on first run and caches it. If offline and uncached, fail with a message naming the cache directory.
- **Stale embeddings cache:** invalidate by content hash of (model_name, sorted node texts). On hash mismatch, re-embed.
- **Viewer with no data:** `index.html` shows a "Run `build_graph_explorer.py` first" message instead of a blank canvas.

We do not try to handle: partial Kuzu failure mid-build (let it crash, the staging dir gets rebuilt next run), or browsers without WebGL2 (we say so in the page).

## Testing

- **Unit:** `score_suspicion.score_all` against a small fixture graph (5 Products, 3 ESHACodes) covering each `disagreement_kind`.
- **Integration:** `build_graph_explorer.py` against a 100-row Kuzu DB extract, verify `data.json` shape and `embedding_suspects.csv` schema match the existing quarantine CSV conventions.
- **Manual sanity:** seed 3 known-bad mappings (e.g. a popcorn product mis-mapped to a milk code), run the pipeline, confirm all 3 surface in the top-50 suspicion ranks. This is the trust-builder before we let it propose at scale.

## Out of scope for this spec (followups)

- Adding a real `Ingredient` node table to the Kuzu schema and using parsed ingredients (vs `Token`) for the graph view.
- Auto-applying remappings from high-confidence `wrong_label` rows.
- Embedding-aware variants of the existing rule-based heal passes.
- Dimensionality reduction with t-SNE (UMAP covers the same need; t-SNE is slower and not as stable across reruns).
